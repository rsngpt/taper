The problem

agent-integrated apps encounter three failure modes that degrade performance on typical budget and mid-range devices (4–6GB RAM) running under tight heap constraints.

SYS-01 // INGEST

heap allocation crashes

parsing large payloads into a DOM tree materializes the entire json string into boxed objects and memory headers. this creates a footprint multiple times the raw size, triggering out-of-memory errors inside default JVM heaps.

SYS-02 // TRIAGE

futile retry loops

a malformed tool call or content policy rejection will never succeed on automatic retries. native retry loops burn user battery and api tokens repeating requests that are destined to fail, while transient failures go untriaged.

SYS-03 // SYNC

connection floods

when an offline device reconnects, queue-draining mechanisms fire dozens of pending message interactions sequentially. this floods weak radio links with redundant HTTP requests when connection strength is lowest.

Three components, one dependency

pure Kotlin, minSdk 24, zero dependencies. use each component alone or together to close the ingest → classify → sync loop.

FP-01 // PARSER

1 · Streaming parser

Define fields of interest to skip unmatched tokens without allocations. Parse large agent responses streamingly with minimal heap footprint.

val result = TaperParser().parse(
  inputStream,
  setOf("model", "usage.total_tokens")
)
val tokens = result
  .firstLong("usage.total_tokens")
FP-02 // CLASSIFIER

2 · Exception classifier

Triage failures programmatically: halt execution on SEMANTIC errors, and queue TRANSIENT errors to resume when conditions allow.

val failure = AgentFailure(429)
val result = classifier.classify(failure)
when (result) {
  TRANSIENT -> queue.enqueue(id, data)
  SEMANTIC  -> surfaceToUser()
}
FP-03 // SYNCER

3 · Offline sync queue

Durable, SQLite-backed storage that survives application crashes. Coalesces queued updates into a single conversation batch upon reconnect.

val queue = SyncQueue.create(context)
queue.enqueue("conv-42", updateJson)
ConnectivityDrainTrigger(
  context, queue, syncer, scope
).start()

How it fits together

failures loop through the classifier; only transient ones re-enter the queue. the ParserEngine seam allows native/ashmem engines in the future.

Honest scope

What v1 is — and deliberately isn't.

  • fully tested: 3 components, 64 unit tests (including a 32MB-heap OOM proof), isolated process benchmarks, and CI on every push.
  • not in v1: JNI native engines (interface only), ML classification (uses deterministic rules), and cloud gateways.
  • measured on-device: every metric is generated by the benchmark harness script inside this repository.
